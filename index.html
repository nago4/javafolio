Javaについて

JavaScriptは、1995年にブレンダン・アイクによって開発されました。
Web開発において重要な役割を果たしており、動的なウェブページやWEBアプリケーション作成するための主要なプログラミング言語のひとつです。

JavaScriptの特徴

  • 動的なウェブページを作成できる
  • イベント駆動型のプログラミング言語
  • クライアントサイドで動作する

JavaScriptを書くときに気を付ける点

  1. 大文字と小文字を区別
  2. セミコロンで区切る
  3. 予約語が存在
  4. strict mode（実行モード）が存在

「1」について

  • var Colar = "red";
  • var colar = "bule";
  • var COLAR = "green";
この「Colar」「colar」「COLAR」は全て別のっ変数として異なる変数として扱われます。
他にも関数名やオブジェクトプロパティ名などの識別子も同様である。

例）var Functoin = 10;
　　　　↑「Function」は予約語なのでエラーとなる。

「2」について

JavaScriptでは、文の最後にセミコロン（;）を付けることが推奨されています。
例）var x = 10;
ただし、セミコロンを付けなくても動作することもあります。
例）var x = 10
　　var y = 20
例）if(x===10){
　　　console.log("xは10です")
　　}

「==」と「===」の違い

「==」よりも「===」のほうが厳密に比較することができます。
例）10 == "10" → true
例）10 === "10" → false

console.logとは

コンピュータ画面上に結果を表示するための命令です。
例）console.log("Hello World");//Hello World

「3」について

予約語
言葉自体で特別な意味を持つ単語です。なので関数名や変数名のような識別子として使用できないです。

予約語の例

  • dreak
    ループやswitch文から抜けるのに使用される。
  • continue
    ループ内の次のインテレーションに進むために使用される。
  • extends
    クラスを継承するために使用される。
  • function
    関数を定義するために使用される。
  • import
    モジュールをインポートするために使われる。
  • new
    オブジェクトを生成するために使用される。

「4」について

strict mode（実行モード）

JavaScriptの実行モードの一つで、コードをより厳密に解析するためのモードです。 コードの先頭で「"use strict";」と記述することでstrict modeを有効にすることができます。

行なわれる制限の例

  • グローバル変数の暗黙的な宣言の禁止
  • 厳密な関数有効範囲のルール
  • 重複する引数名やプロパティ名の禁止
  • eval関数（文字列を評価しその値を返す）の厳密な動作
  • セキュリティの強化
